## 1️⃣ Nginx 是什么？

* **Nginx**（发音：engine-x）是一款**开源的高性能 Web 服务器**。
* 它可以做 **网站服务器、静态文件服务器、反向代理、负载均衡** 等功能。
* 简单理解：

  * 它就像一名 **前台接待员**，帮你把用户请求分发给不同的“部门”（Node 后端、静态前端文件等）。

---

## 2️⃣ Nginx 的主要作用

| 作用         | 举例解释                                            |
| ---------- | ----------------------------------------------- |
| 静态文件服务器    | 你的网站前端 HTML/CSS/JS 文件，用户访问直接返回，不需要 Node 处理      |
| 反向代理       | 用户请求 `/api` → Nginx 把请求转发给 Node 后端处理，然后把结果返回给用户 |
| 负载均衡       | 有多台 Node 后端服务器时，Nginx 可以把请求平均分配，保证不堵车           |
| HTTPS / 安全 | 可以配置证书，统一管理 HTTPS，不用每个 Node 去处理                 |

---

## 3️⃣ 反向代理是什么？

* **正向代理**：你用电脑访问互联网，请求先给代理服务器，再由它去访问目标网站（像 VPN）。
* **反向代理**：用户请求网站 → 先到 Nginx → 再由 Nginx 转发给真正的服务器（Node 后端），最后结果再返回给用户。

用图理解：

```
浏览器发请求 → Nginx → Node 后端 → 数据库
         ←——←——←——返回结果
```

💡 好处：

1. Node 后端可以只处理 API，不直接面对用户。
2. 前端静态文件可以直接用 Nginx 处理，更快。
3. 可以轻松切换 Node 后端或多台服务器。

---

## 4️⃣ 举例：前端 + Node + Nginx

假设你有一个项目：

* 前端 React 构建后：`build/index.html`、`build/static/js/...`
* Node 后端：提供 API `/api/users`
* 用户访问网址：

**用户访问**：

```
http://你的域名/
```

**流程**：

1. 请求到 Nginx
2. Nginx 判断：

   * `/api` → 转发到 Node 后端 3000 端口
   * 其它 → 直接返回前端 `index.html` 或静态文件
3. Node 后端处理业务逻辑 → 查询数据库 → 返回 JSON
4. Nginx 把结果返回给用户

---

## 5️⃣ 为什么 Node 项目上线一般都用 Nginx

* Node.js 本身也可以直接监听 80/3000 端口，但有几个问题：

  1. 静态文件效率低 → Nginx 更快
  2. 没有内置 HTTPS → Nginx 统一处理
  3. 可以做反向代理 / 负载均衡 → 扩展性强
  4. 稳定性好 → Node 崩溃不会直接影响静态文件访问

---

## 6️⃣ 简单总结

| 名词      | 小白理解                          |
| ------- | ----------------------------- |
| Nginx   | 前台接待员，负责接收所有用户请求，分发到 Node 或前端 |
| 静态文件服务器 | 直接把前端文件返回给用户                  |
| 反向代理    | 帮 Node 隐藏真实地址，转发 API 请求       |
| 负载均衡    | 多台 Node 服务器平均分配请求，防止堵车        |

---

💡 **最简单比喻**：

* **浏览器** → 客户
* **Nginx** → 前台接待员
* **Node 后端** → 内勤部门
* **数据库** → 仓库

用户进门（访问网站） → 前台分配 → 内勤处理 → 仓库取数据 → 返回给用户。
