## 一、Node.js 为什么不能直接部署就算完？

Node.js 本身是一个 **JavaScript 运行环境**，你在服务器上直接运行：

```bash
node app.js
```

你的项目确实能跑起来，但存在几个问题：

1. **程序崩溃后不会自动重启**

   * Node.js 出现未捕获异常或错误时，进程会直接退出
   * 用户访问接口就会失败

2. **不能后台运行**

   * 直接 `node app.js`，终端关闭后程序就挂了
   * 需要保持一个终端一直开着，或者用 `&` 弹后台，但管理不方便

3. **缺少日志管理**

   * 没有方便的方式记录 stdout/stderr（请求日志、报错日志）
   * 排查问题不方便

4. **不能轻松管理多个应用**

   * 如果你有多个 Node 项目，直接 node 启动，管理和重启会很麻烦

---

## 二、PM2 的作用

PM2 是 Node.js 官方推荐的 **进程管理工具**，解决了上面问题：

| 功能       | 说明                            |
| -------- | ----------------------------- |
| 🟢 后台运行  | Node 程序可以脱离终端运行，不会因 SSH 断开而挂掉 |
| 🔄 自动重启  | 程序崩溃或报错时自动重启，保证服务在线           |
| 📊 日志管理  | 自动记录 stdout 和 stderr，方便排查问题   |
| ⚙️ 多应用管理 | 可以同时管理多个 Node 项目，统一启动/重启/停止   |
| 🚀 开机自启  | 可以让服务器重启后 Node 程序自动启动         |

---

### 常用命令

```bash
# 安装 PM2
npm install -g pm2

# 启动项目
pm2 start app.js --name myapp

# 查看状态
pm2 list

# 查看日志
pm2 logs myapp

# 重启项目
pm2 restart myapp

# 停止项目
pm2 stop myapp

# 设置开机自启
pm2 save
pm2 startup
```

---

## 三、总结

| 直接用 node  | 使用 PM2     |
| --------- | ---------- |
| 程序挂了就停    | 程序挂了自动重启   |
| 终端关闭，程序挂掉 | 脱离终端后台运行   |
| 日志乱       | 自动管理日志     |
| 多项目管理困难   | 一条命令管理多个项目 |

✅ 所以，**Node 项目部署上线一般都推荐使用 PM2**，这是服务器稳定运行的最佳实践。
