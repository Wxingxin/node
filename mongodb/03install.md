# 🟢 **mongod 和 mongos 到底是什么？有什么区别？**

在 MongoDB 的 `bin` 文件夹里，最重要的两个可执行文件就是：

* `mongod`
* `mongos`

它们的作用完全不同。

---

# 🟩 **1. mongod —— MongoDB 的核心数据库服务（必用）**

你要启动 MongoDB，必须启动的就是：

```
mongod.exe
```

### ✔ 作用

`mongod` 是 **数据库本体**，负责：

* 保存数据
* 执行查询
* 管理数据库文件（存储引擎）
* 提供 27017 端口监听
* 处理客户端请求

简单说：

> **mongod = 真正的 MongoDB 数据库。
> 你平时 99% 的时候只用它。**

---

# 🟧 **2. mongos —— 分片集群的路由服务（普通用户用不到）**

`mongos` 只有一种场景需要：

👉 **你要搭建 MongoDB 分片集群**

假如你做的是 **大型分布式 MongoDB 集群**，使用多台服务器做分片，才会用到 `mongos`。

### ✔ 作用

`mongos` 是 **路由器 (router)**，负责：

* 接收客户端请求
* 决定把数据写到哪个分片（shard）
* 负责分布式数据管理的转发
* 聚合多个分片上的数据

你可以理解为：

> **mongos = MongoDB 分片集群的“网关” / “路由器”。
> 不是数据库，不存数据，也不写磁盘。**

---

# 🧩 简单理解：

| 文件         | 作用              | 是否必须    |
| ---------- | --------------- | ------- |
| **mongod** | 真正的 MongoDB 数据库 | ✔ 必须    |
| **mongos** | 分片集群的路由管理器      | ✖ 平时用不到 |

---

# 🧪 举例让你完全明白：

### 你开发普通项目（Node.js、Express、Python、React）

你只需要：

```
mongod
```

### 你做高并发、大型集群，需要分片

你才需要：

```
mongos（多个）
mongod（多个）
config server（多个）
```

---

# 🎯 你当前，只需要记住一句：

> **你只需要用 mongod。
> mongos 对你完全没用（除非你做分片集群）。**
